# 研究提案書: 位置情報マッチングアプリにおけるボランティアベース情報共有の成立条件

## 1. 研究背景

### 1.1 現代社会における情報の非対称性

現代社会において、リアルタイムな現地情報を得ることは依然として困難である。特に以下のような状況では、Web上の情報だけでは不十分である。

- 店舗の営業状況 (急な休業、混雑状況)
- 自然環境の状態 (海の波の高さ、紅葉の状況)
- 商品の在庫状況 (人気商品の入荷状況)
- イベント会場の混雑状況

これらの情報は「今その場所にいる人」しか正確に把握できない。

### 1.2 既存サービスの課題

SNSや口コミサイトでは、以下の問題がある。

- **情報の鮮度**: 投稿された情報が古く、現在の状況を反映していない
- **情報の信頼性**: 主観的な情報や不正確な情報が混在
- **情報提供のモチベーション**: ユーザーが積極的に情報を投稿する動機が弱い

### 1.3 本研究の意義

本研究では、**金銭的インセンティブなし**のボランティアベースで、以下を検証する。

1. ボランティアベースで情報提供者が集まるか
2. リアルタイムな現地情報のマッチングが成立するか
3. C2C (個人間) の助け合い文化が形成されるか

---

## 2. 研究目的

### 2.1 主目的

**「ボランティアベースの位置情報マッチングアプリにおいて、ユーザー間の情報共有がどの程度成立するかを検証する」**

### 2.2 副次的目的

1. 回答率に影響を与える要因を特定する
2. 完了率に影響を与える要因を特定する
3. ユーザーの行動パターン (依頼側 vs 回答側) を分析する
4. ボランティアベースでのモチベーション要因を明らかにする

---

## 3. 研究課題 (Research Questions)

### RQ1: ボランティアベースの成立可能性
**「金銭的インセンティブなしで、ユーザーは情報提供するか?」**

**仮説**: 一定数のユーザーはボランティアで情報提供する意欲を持つ。

**検証方法**:
- 回答率の測定 (目標: 40%以上)
- 回答速度の測定 (目標: 48時間以内)
- ユーザーアンケート (情報提供の動機)

### RQ2: マッチング成立の条件
**「どのような依頼がマッチングしやすいか?」**

**仮説**: 明確で具体的な依頼、写真付き依頼、簡単な依頼がマッチングしやすい。

**検証方法**:
- 依頼内容 (文字数、明確さ、参考写真の有無) と回答率の関係性分析
- ロジスティック回帰分析

### RQ3: ユーザー行動の分類
**「ユーザーは依頼側と回答側のどちらに偏るか?」**

**仮説**: 多くのユーザーは依頼側に偏り、一部のヘビーユーザーが回答側として活動する。

**検証方法**:
- ユーザーの依頼回数 vs 回答回数の分布分析
- クラスタリング分析 (K-means)

### RQ4: 信頼性の担保
**「完了率はどの程度か? また、低品質な回答はどの程度発生するか?」**

**仮説**: 完了率は70%以上であり、ボランティアベースでも一定の品質が保たれる。

**検証方法**:
- 完了率の測定
- 完了までの時間分析
- ユーザーアンケート (回答品質への満足度)

## 4. 研究手法

### 4.1 研究デザイン

**実験的研究** (Experimental Study)

- プロトタイプアプリを実際にリリースし、実データを収集
- 定量的データ分析 + 定性的ユーザーインタビュー

### 4.2 データ収集方法

#### 4.2.1 定量データ
以下のデータをFirestore (データベース) から抽出する。

| データ項目 | 説明 |
|-----------|------|
| ユーザー数 | 登録ユーザー数、アクティブユーザー数 |
| 依頼投稿数 | 総依頼数、カテゴリ別依頼数 |
| 回答投稿数 | 総回答数、ユーザー別回答数 |
| 回答率 | 依頼に対して回答が付いた割合 |
| 平均回答時間 | 依頼投稿から回答までの時間 |
| 完了率 | 回答が完了とされた割合 |
| ユーザーの役割分布 | 依頼のみ / 回答のみ / 両方のユーザー比率 |

#### 4.2.2 定性データ
以下の方法でユーザーの意見を収集する。

- **アプリ内アンケート**: 利用後の満足度、改善点、情報提供の動機
- **ユーザーインタビュー**: 5〜10名を対象に、利用動機や課題を深掘り

### 4.3 実験期間

**6週間** (プロトタイプリリースから検証まで)

| 週 | 内容 |
|----|------|
| 1週目 | リリース、初期ユーザー獲得 (目標: 20人) |
| 2〜4週目 | データ収集期間 (目標: 50人、30件の依頼) |
| 5週目 | ユーザーアンケート・インタビュー実施 |
| 6週目 | データ分析、論文執筆 |

### 4.4 分析手法

#### 4.4.1 記述統計
- 平均、中央値、標準偏差
- 分布図 (ヒストグラム、箱ひげ図)

#### 4.4.2 推測統計
- 相関分析 (Pearson, Spearman)
- t検定、ANOVA (グループ間比較)
- 回帰分析 (線形回帰、ロジスティック回帰)

#### 4.4.3 クラスタリング
- K-means法によるユーザー分類 (依頼者型、回答者型、両用型)

## 5. 期待される成果

### 5.1 学術的貢献

- **ボランティアベース情報共有の知見**: 金銭的インセンティブなしでの成立条件を実証
- **内発的動機の分析**: なぜ人は無償で情報提供するのかを明らかにする
- **位置情報マッチングの可能性**: 新しい情報流通モデルの提案

### 5.2 実務的貢献

- **企業への応用**: 店舗チェーンが顧客に現地調査を依頼するモデルへの展開
- **自治体への応用**: 災害時の現地情報収集システムへの応用
- **NPO/NGOへの応用**: ボランティアベースの情報共有プラットフォーム

### 5.3 社会的貢献

- 情報の非対称性を解消し、より正確な現地情報へのアクセスを提供
- 移動が困難な人でも、現地情報を得られる手段を提供
- 助け合い文化の醸成

## 7. 研究の限界と今後の課題

### 7.1 研究の限界

- **サンプルサイズ**: プロトタイプのため、ユーザー数が限定的 (目標50人)
- **地域の偏り**: 初期段階では特定地域 (大学周辺) に集中する可能性
- **実験期間**: 6週間の短期検証のため、長期的な行動変化は不明
- **セレクションバイアス**: 協力的なユーザーのみが参加する可能性

### 7.2 今後の課題

- **不正行為の防止**: 虚偽情報への対策
- **品質管理**: 低品質な回答や不適切な写真への対処
- **法的課題**: プライバシー保護、私有地への侵入リスク
- **スケーラビリティ**: ユーザー数が増加した際の持続可能性
- **ゲーミフィケーション**: 将来的なバッジやランキング機能の効果検証

## 8. 倫理的配慮

### 8.1 個人情報保護

- ユーザーの位置情報は暗号化し、第三者に提供しない
- 利用規約で、データの研究利用について明記し、同意を得る
- 個人を特定できる情報は匿名化して分析

### 8.2 プライバシー保護

- 回答写真に個人の顔や車のナンバープレートが映らないよう注意喚起
- 不適切な写真が投稿された場合、運営側で削除

### 8.3 利用規約

- 私有地への無断侵入を禁止
- 虚偽情報の投稿を禁止
- 違反者はアカウント停止措置

## 11. 研究計画書のまとめ
